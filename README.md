# MetaBlog — Блог на Next.js

Современный и адаптивный блог, разработанный на базе фреймворка **Next.js (App Router)** с использованием **TypeScript**. Дизайн интерфейса реализован по мотивам макетов из Figma с применением Pixel-Perfect подхода.

## 🌟 Основные возможности

- **App Router:** Использование современных серверных компонентов (RSC) Next.js для быстрой отрисовки страниц и лучшего SEO.
- **Тёмная и светлая тема:** Плавное ручное переключение тем (через иконку в Header) с сохранением выбора пользователя, реализованное с помощью библиотеки `next-themes` и CSS-переменных. Никакого мерцания стилей (FOUC).
- **Динамический Hero-блок:** Главный (самый большой) пост на странице генерируется случайным образом при каждом обновлении страницы. Картинка и заголовок полностью кликабельны для перехода к чтению.
- **Пагинация:** Постраничный вывод постов на главной странице с удобной навигацией.
- **Случайный пост (Random Post):** Кнопка и пункт меню для чтения случайной статьи. Механизм защищен от выдачи 404 ошибок, а переход осуществляется плавно.
- **Адаптивный дизайн (Responsive):** Сайт корректно отображается как на десктопных, так и на мобильных устройствах, используя Media Queries и CSS Grid / Flexbox.
- **Уникальные пользователи:** Чтобы лента выглядела разнообразно, аватары и имена авторов привязываются математически к ID поста, исключая дублирование одних и тех же авторов.
- **Оптимизация производительности (Core Web Vitals):** Максимально оптимизирована загрузка фотографий (использование `<Image>` с поддержкой форматов `avif`/`webp` и тонкими настройками `sizes` атрибутов). Убраны лишние рендеры.
- **Чистый код:** Весь код полностью проверен Typescript и строгими правилами ESLint (включая `react-hooks/purity`), очищен от мусора и содержит подробные комментарии на русском языке.

## 📂 Структура проекта

```text
src/
├── app/                  # Маршруты и страницы Next.js App Router
│   ├── globals.css       # Глобальные стили (переменные цветов и темы)
│   ├── layout.tsx        # Корневой шаблон: подключает шрифты, Header, Footer и ThemeProvider
│   ├── page.tsx          # Главная страница: список постов, динамический Hero-блок, баннеры (отключены)
│   ├── contact/          # Страница контактов
│   └── posts/
│       ├── [id]/         # Динамический роут: детальная страница отдельного поста
│       └── random/       # API-роут: серверный редирект на случайную статью (без кэша)
├── components/           # Изолированные UI-компоненты
│   ├── Footer/           # Подвал сайта (навигация, подписка - скрытые нерабочие ссылки)
│   ├── Header/           # Шапка сайта (логотип, поиск, кнопка темной темы с hydration-fix)
│   ├── Pagination/       # Компонент постраничной навигации
│   ├── PostCard/         # Карточка поста (превью статьи для сетки)
│   └── ThemeProvider/    # Обёртка конфигурации next-themes
├── lib/                  # Утилиты
│   └── api.ts            # Сервисный слой для дата-фетчинга внешнего API
└── types/
    └── post.ts           # TypeScript типы моделей (Post, PaginationResult)
```

## 🚀 Развёртывание и запуск локально

1. Установите необходимые зависимости:

   ```bash
   npm install
   ```

2. Запустите проект в режиме локальной разработки:

   ```bash
   npm run dev
   ```

3. Для проверки production-сборки (включая строгую проверку типов и линтеров):

   ```bash
   npm run build
   ```

4. Откройте [http://localhost:3000](http://localhost:3000) в вашем браузере.

## 🔧 Ключевые архитектурные решения

- **Темы оформления**: Из-за того, что классический `@media (prefers-color-scheme: dark)` не позволяет пользователю вручную переключать тему (только через настройки ОС), был внедрен модуль `next-themes`. Все цвета зависят от атрибута `:root[data-theme='dark']`.
- **API и Random Post ограничитель**: Так как используется внешний API _JSONPlaceholder_, база ограничена строго 100 постами. Логика генерации случайного поста ограничена диапазоном 1-100, чтобы не допускать 404 ошибок.
- **Оптимизация изображений**: В `next.config.ts` настроены `remotePatterns` и форматы `image/avif`, `image/webp`. Встроенный компонент `<Image />` использует атрибуты `fill` и `sizes` для загрузки оптимальных микроминиатюр на мобильных устройствах, сохраняя трафик пользователей.
- **Обход ESLint Pure Hooks**: Поскольку в Server Components (`page.tsx`) используется случайная генерация (`Math.random()`), логика вынесена вне тела React-функции, чтобы не нарушать принципы чистоты рендеринга (айдемпотентность).
- **Отсутствие битых ссылок**: Нерабочие разделы интерфейса (социальные сети, пустые категории и баннеры) либо отключены через `<span>` вместо `<Link>`, либо закомментированы до реальной интеграции бекенда.
